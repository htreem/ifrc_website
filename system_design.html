System Design

System Architecture

Our system architecture consists of 5 main steps.

Step 1: Data Acquisition

We first acquired the necessary input data from the 4 data sources (EM-DAT, Desinventar, IDMC, IFRC).

Step 2: Data Cleaning

We then resolved issues regarding missing data, outliers, inconsistently named fields, null values and working concurrently with group 5 to ensure our data sets are compatible with theirs.

Step 3: Data Shaping

We then merge data from all the data sets and decompose each hazard from every source down to the actual data we need - hazard types, the hazard countries ISO, the hazard effects and the hazard date before iteratively inserting them into our unique tree structure.

Step 4: Data Analysis

After we shaped the data in the form of our tree, we then started using it to get more useful data in the form of percentages and trends.

Step 5: Data Visualisation

We then generated tables for the IFRC by having the data we calculated and the data from our unique tree structure moved to a 2D array.

Raw Data Storage

There’s a root node which is connected to each country which acts as a root node for their own separate trees, essentially giving us independent trees for each country, then each root node has 4 grandchildren (month, year, hazard profile, hazard cluster).

Processed Data Storage

The final set of data including both raw and processed data will be moved to a singular data frame in the form of a 2D array. This is the final deliverable that the IFRC are expecting.

Design Patterns and Practices

Pair programming and test driven development practices are employed to minimise bugs and errors.
