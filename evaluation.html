Critical Evaluation of the Project

Functionality

We have designed a programme which scrapes raw data from four files (EM-DAT, Desinventar, IDMC’s GIDD and IFRC) and merges them into a single dataset (a unique tree structure). The code then analyses the new dataset to produce useful information in the form of percentages and trends. The code then allocates the processed and raw data into a single data frame (a 2d ARRAY) which is our final deliverable.

Stability

Our code is stable and reproducible as long as we continue to use the same original data sources the formatting of the original data sources remain unchanged.

Efficiency and Maintainability

Our code is very simple, so even though we did not take efficiency into account as it was not a priority, our code still doesn’t take an egregious amount of time to run. It is also very easy to maintain as we do not expect it to require any modifications anywhere soon.

Compatibility

Our code should be compatible across any device as it was coded on python and doesn’t take much to run or implement.

Project Management

Project management was not an issue as we worked well together as a team and fulfilled all our roles. The roles we fulfilled were mainly delegated by our team lead Dylan Penney after we let him know of each of our strength and weaknesses
