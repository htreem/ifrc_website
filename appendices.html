Deployment Manual

Step 1

Download dependencies any way you like.

Step 2

Run the data-downloader repository to obtain data from 4 different sources.

Step 3

Run the data-shaping repository to shape the raw data into our unique tree structure.

Step 4

Run the data-processor repository to obtain useful statistical data in the form of percentages and trends

Step 5

Run the data-visualiser repository to move all the processed and raw data we need into a single data-frame (2D array). This repository should return the 2D array as our final deliverable.

Legal Issues and Processes

On our end, we have the following liabilities:

To complete and deliver a project that satisfies the agreed requirements, as per discussed with our project partner, by the set deadline (29th May). The final deliverable of the project includes a singular data frame which holds raw and processed data from 4 sources (EM-DAT, Desinventar, IDMC’s GIDD, IFRC).

Distribution of materials and data, provided by our clients, without their consent should be avoided.

Making our work public without clients’ consent should be avoided. The work here mainly remains in the GitHub we made designated to this project. The IFRC data source, which the client has asked us to keep private, gives us more reason to not share the GitHub carelessly.

Intellectual Property Type

After discussing it together, we have decided that the intellectual property type that suited our project the most was Copyright © and not a Trademark or Patent. This is because we haven’t given our project a name, as we didn’t need one due to us not marketing it, and us creating our code implementing pre-existing technologies and methods. Hence by definition, this code is classed as intellectual property (IP) of the creator(s) (UCL Group 4).

We have also implemented various open-source libraries in our code including but not limited to pandas, selenium, and pycountry. They consisted of the necessary functionalities which helped us design our code for data scraping and analysis. Since they are classed as open source materials, they are protected under an open source license, hence they are free to use to our own benefit.
